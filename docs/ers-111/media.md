# ERS-111 Media Research

The key question is whether this first-generation Aibo can be given new
boot media in a safe, repeatable way similar to later ERS-7 work.

The main uncertainty is that early Aibo hardware appears to predate the
standard later Memory Stick workflow, so the ERS-111 may require a
different removable-media path and different staging rules.

## Current Assumptions

- the original working media must be preserved before any experiments
- all testing should happen on sacrificial duplicate media
- later ERS-7 Memory Stick procedures should be treated as reference only
- the local `r-code/` package is evidence of a real `ERS-110/111` Memory Stick programming workflow

## Tasks

- identify the exact ERS-111 removable media in use on this unit
- document whether it is native Memory Stick, PC Card, or an adapter-based arrangement
- record media brand, capacity, labels, and adapter details
- check whether the real unit and local R-Code materials agree on `8MB` or `16MB` media expectations
- note any limits that may matter for duplicate media selection
